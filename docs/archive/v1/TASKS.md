## 코드 규칙

### 커밋 컨벤션

- `feat(태스크ID): 설명` — 새 기능
- `fix(태스크ID): 설명` — 버그 수정
- `chore: 설명` — 설정/유지보수
- `refactor: 설명` — 리팩토링
- `docs: 설명` — 문서
- `test: 설명` — 테스트

### 브랜치 네이밍

- 기능: `feat/P{phase}-{기능명}`
- 버그: `fix/P{phase}-{설명}`
- 핫픽스: `hotfix/{설명}`

### 완료 정의 (Definition of Done)

- [ ] 코드 작성 완료 (태스크 완료 기준 충족)
- [ ] 빌드 통과 (`npm run build`)
- [ ] 린트 통과 (`npm run lint`)
- [ ] 브라우저 테스트 통과
- [ ] 문서 업데이트 (TASKS.md, ROADMAP.md)
