# 상품권 교환권 플랫폼 PRD (Product Requirements Document)

> 작성일: 2026년 2월
> 최종 수정: 2026년 2월 27일
> 버전: v1.0

---

## 0. 제품 개요

### 0-1. 배경 및 목적

상품권/교환권을 온라인으로 구매하고, 핀(PIN) 번호를 안전하게 수령하며, 다른 회원에게 선물할 수 있는 플랫폼입니다. 카카오 선물하기와 유사한 선물/기프트 감성의 사용자 경험을 제공하되, 상품권 핀 번호 중개에 특화된 서비스입니다.

### 0-2. 핵심 가치

- 구매 즉시 문자(SMS)로 바우처 URL 1개 발송, 어디서든 핀 번호 확인 가능 (수량 N개 구매 시 핀 N개를 한 화면에서 확인)
- 임시 비밀번호 + 사용자 비밀번호 2단계 보안으로 핀 번호 보호
- 회원 간 상품권 선물 기능 (새 URL 발급 방식)
- 수수료 포함/별도 선택으로 유연한 가격 구조

### 0-3. 타겟 사용자

| 사용자 유형 | 설명 | 핵심 니즈 |
|------------|------|----------|
| 구매자 | 상품권을 본인 사용 또는 선물 목적으로 구매하는 일반 소비자 | 간편한 구매, 빠른 핀 수령, 안전한 보관 |
| 선물 수신자 | 다른 회원으로부터 상품권을 선물받는 사용자 | 쉬운 수령 과정, 핀 확인 편의성 |
| 관리자 | 상품권 등록, 주문/회원/재고를 관리하는 운영자 | 효율적인 운영, 빠른 검색/필터, 재고 관리 |

### 0-4. MVP 범위

**포함 (MVP v1.0)**

- 회원가입/로그인 (아이디/비밀번호, 본인인증)
- 상품권 카테고리 조회 및 구매 (워너페이먼츠 PG)
- 바우처 URL 발급 + 임시 비밀번호 SMS 발송 (알리고)
- 비밀번호 설정 → 핀 번호 해제 (수수료 포함/별도)
- 회원 간 선물하기 (새 바우처 URL 발급)
- 결제취소/환불 (비밀번호 설정 전)
- 관리자 대시보드, 주문/회원/상품/핀/선물/취소 관리
- 관리자 필터/검색/CSV 내보내기

**미포함 (향후 확장)**

- 소셜 로그인 (카카오, 네이버, 구글)
- 장바구니 기능
- 쿠폰/프로모션 시스템
- 다국어 지원
- 네이티브 앱 (iOS/Android)
- 관리자 권한 분리 (슈퍼관리자/일반관리자)
- B2B 대량 구매 기능
- 통계/리포트 고도화 (BI 연동)

### 0-5. 디자인 가이드라인

- **이모지 사용 금지**: 모든 UI, SMS, 알림 메시지에 이모지 금지. 아이콘은 lucide-react 사용.
- **톤앤매너**: 선물/기프트 감성의 따뜻하고 깔끔한 디자인 (카카오 선물하기 참고)
- 상세 디자인 요구사항은 [10-6. 디자인] 참조

## 3. 바우처(교환권) 영역 상세 화면 정의

> ※ 바우처 페이지는 문자로 받은 URL을 통해 접근하는 독립적인 페이지입니다. 사이트 GNB 없이 단독으로 동작합니다.

### 3-1. 바우처 메인

- **URL**: `/v/[code]`
- **설명**: 문자로 받은 URL 첫 진입 화면. **최초 접속 시 보이스피싱 주의 화면을 먼저 표시하고, 확인 후** 20분 타이머와 임시 비밀번호 입력. 비밀번호 설정 전까지 결제취소 가능.
- **타이머 동기화**: 타이머는 서버 기준 `temp_password_expires_at` 값으로 계산합니다. 프론트엔드는 접속 시 서버에서 남은 시간을 받아 카운트다운합니다. 새로고침/재접속 시에도 정확한 잔여 시간이 표시됩니다.
- **재발행 규칙**: 재발행 버튼은 타이머 만료 후에만 노출됩니다. 만료 전에는 문자를 다시 확인하도록 안내합니다. 재발행은 최대 5회까지 가능하며, 초과 시 고객센터 문의 안내가 표시됩니다.
- **선물받은 바우처**: 선물로 받은 바우처는 **새로운 URL(새 바우처 코드)**로 발급됩니다. 기존 보낸 사람의 URL과 다릅니다. 결제취소 버튼이 노출되지 않으며, 보낸 사람 정보가 상단에 표시됩니다.

| 구성 요소 | 타입 | 설명 | 비고 |
|-----------|------|------|------|
| 보이스피싱 주의 화면 | FullScreen Overlay | 최초 접속 시 전체 화면 보이스피싱 주의 안내. 3가지 사기 유형: ①카카오톡 해킹 사칭, ②경찰/검찰 사칭, ③금융기관 사칭 | 세션/쿠키 기반으로 해당 바우처 코드에 대해 1회만 표시. 새로고침/재접속 시 다시 표시하지 않음 |
| "확인하였습니다" 버튼 | Button (CTA) | 주의사항 확인 후 메인 화면으로 진행 | 확인 전까지 다른 UI 접근 불가 |
| 상품권 정보 | Card | 상품권 이름, 가격, 수량 표시 | 구매 내역 요약. 수량 N개 구매 시 "N개" 표시 |
| [선물받은 경우] 보낸 사람 표시 | Text (accent) | "OOO님이 보낸 선물" | 선물받은 바우처에만 노출 |
| 20분 타이머 | Countdown Timer | 남은 시간 MM:SS 형태로 실시간 표시 | 빨간색 경고 (5분 이하) |
| 임시 비밀번호 입력 | Input (3자리) | 문자로 받은 영어+숫자 3자리 입력 | 입력 시 자동 포커스 이동 |
| 확인 버튼 | Button | 임시 비밀번호 검증 | 5회 실패 시 잠금 |
| **결제취소 버튼** | **Button (destructive)** | **결제 취소 요청 페이지로 이동** | **직접 구매 + 비밀번호 설정 전에만 노출. 선물받은 바우처에는 미노출** |
| 취소 안내 텍스트 | Text (muted) | 비밀번호 설정 후에는 결제 취소가 불가합니다 | 결제취소 버튼과 함께 노출 |

| 재발행하기 버튼 | Button | 타이머 만료 후 노출. 새 임시 비밀번호 문자 발송 | 클릭 시 새 비밀번호 발급 + 타이머 리셋. **최대 5회 제한** |
| 재발행 횟수 초과 안내 | Alert (destructive) | "재발행 횟수를 초과했습니다. 고객센터에 문의해주세요" | 5회 초과 시 재발행 버튼 대신 노출 |

### 3-5. 선물하기

- **URL**: `/v/[code]/gift`
- **설명**: 보유한 상품권을 다른 회원에게 선물 전송
- **로그인 필요**: 회원 검색을 위해 로그인 상태여야 합니다. 미로그인 시 로그인 페이지로 이동 후 복귀.
- **선물 후 처리**: 선물 전송 완료 시 기존 바우처는 비활성화되고, 수신자에게 **새로운 바우처 코드(새 URL)**가 발급됩니다. 포함된 모든 핀(N개)이 새 바우처로 함께 이동합니다. 기존 URL에 접속하면 "선물이 완료된 상품입니다" 메시지가 표시됩니다.

| 구성 요소 | 타입 | 설명 | 비고 |
|-----------|------|------|------|
| 로그인 확인 | 자동 체크 | 미로그인 시 로그인 페이지로 리다이렉트 | 로그인 후 선물하기 페이지로 복귀 |
| 회원 검색 Input | SearchInput | 아이디로 회원 검색 | 실시간 자동완성 또는 검색 버튼 |

| 자기 자신 선물 방지 | Validation | 본인 아이디 선택 시 에러 메시지 | "자기 자신에게는 선물할 수 없습니다" |
| 선택된 수신자 정보 | Card | 선택한 회원 아이디 표시 | 변경/취소 가능 |
| 선물 메시지 (선택) | Textarea | 받는 사람에게 전할 메시지 | 최대 100자, 선택 입력 |
| 결제취소 불가 안내 | Alert (destructive) | "선물 전송 후에는 결제 취소가 불가합니다." | 선물하기 버튼 상단에 항상 표시, 빨간 배경 강조 |

| 선물하기 버튼 | Button (CTA) | 선물 전송 실행 | 전송 후 문자 발송 |
| 선물 완료 화면 | Card + Text | "선물이 전송되었습니다. 수신자에게 문자가 발송되었습니다." | 마이페이지/메인 이동 버튼 |

### 3-6. 핀 번호 확인

- **URL**: `/v/[code]/pin`
- **설명**: 핀 번호 해제 프로세스. 수수료 포함/별도에 따라 분기. **바우처에 포함된 모든 핀 번호를 한 화면에서 리스트로 표시합니다** (예: 5개 구매 시 핀 5개 동시 표시).
- **재접속 동작**: 핀 번호를 이미 확인한 사용자가 재접속 시, 비밀번호 재입력 후 다시 핀 번호를 노출합니다 (반복 조회 가능).
- **핀 해제 단위**: 바우처 단위로 일괄 해제됩니다. 개별 핀 단위 해제는 불가합니다.

## 8. 주요 비즈니스 플로우 요약

### A. 구매 플로우

메인/카테고리 → 상품 상세 (수수료 포함/별도 선택) → 주문/결제 (워너페이먼츠) → 주문 완료 → 알리고 문자 발송 (URL + 임시 비밀번호)

### B. 비밀번호 설정 플로우

URL 접속 → **보이스피싱 주의 확인** → 20분 타이머 표시 → 임시 비밀번호 입력 → 4자리 비밀번호 설정 → 완료 → 선물하기/핀 해제 선택 화면

### C. 타이머 만료 플로우

20분 경과 → "비밀번호 설정 시간이 초과되었습니다." 메시지 표시 → 재발행하기 버튼 → 새 임시 비밀번호 문자 발송 → 타이머 리셋 → 처음부터 다시. **재발행은 최대 5회까지 가능.** 5회 초과 시 "재발행 횟수를 초과했습니다. 고객센터에 문의해주세요" 표시. 타이머는 서버 기준 `temp_password_expires_at`으로 계산하며, 페이지 이탈 후 재접속 시에도 정확한 잔여 시간 동기화.

### D. 핀 해제 플로우 (수수료 포함)

핀 번호 해제 클릭 → 4자리 비밀번호 입력 → 인증 성공 → 핀 번호 전체(N개) 즉시 표시 (바우처 단위 일괄 해제)

### E. 핀 해제 플로우 (수수료 별도)

핀 번호 해제 클릭 → 4자리 비밀번호 입력 → 인증 성공 → "수수료 결제 후 핀 번호가 노출됩니다" 안내 메시지 (수수료 = 단가 × 수량) → 수수료 결제 (워너페이먼츠) → 핀 번호 전체(N개) 표시

### F. 선물하기 플로우

선물하기 클릭 → **결제취소 불가 안내 표시** → 회원 아이디 검색 → 수신자 선택 → 확인 모달 (선물/결제 취소 불가 재안내) → 선물 전송 → **기존 바우처 비활성화 + 새 바우처 코드(새 URL) 생성 + 포함된 핀 N개 모두 새 바우처로 이동** → 수신자에게 문자 발송 (보낸 사람, 상품명, 가격, 수량, **새 URL**, 새 임시 비밀번호) → 기존 URL 접속 시 "선물이 완료된 상품입니다" 표시

### G. 결제취소 플로우

바우처 URL 접속 (비밀번호 설정 전) → 결제취소 버튼 클릭 (별도 인증 불필요) → 취소 사유 입력 → 확인 모달 → PG 취소 API 호출 (결제 금액 전액 환불: 수수료 포함 주문은 상품+수수료, 별도 주문은 상품 금액만) → 취소 완료 → 구매자에게 취소 문자 발송 → 핀 번호 N개 전체 재고 복구 (주문 전체 취소만 가능, 부분 취소 불가)

### H. 결제취소 불가 조건

비밀번호 설정 완료 후에는 결제취소 불가. 핀 해제 완료 후에는 결제취소 불가. **선물 전송 완료 후에는 결제취소 불가 (선물하기 화면과 확인 모달에서 사전 안내 필수).** 이 경우 고객센터를 통한 별도 처리 안내.

### I. 핀 번호 재조회 플로우

이미 핀 해제된 바우처 URL 재접속 → 바우처 액션 화면 ("핀 번호 다시 보기" 버튼) → 4자리 비밀번호 재입력 → 인증 성공 → 핀 번호 전체(N개) 즉시 표시 (수수료 재결제 없음)

### J. 선물 수신자 플로우

선물 수신 문자 수신 (새 URL 포함) → **새 URL** 접속 → **보이스피싱 주의 확인** → 20분 타이머 시작 → 새 임시 비밀번호 입력 → 4자리 비밀번호 설정 → 바우처 액션 (선물하기/핀 해제) → 이후 동일 플로우. 보낸 사람의 기존 URL과는 완전히 다른 URL.

### K. 수수료 결제 실패 플로우

수수료 결제 버튼 클릭 → 워너페이먼츠 결제창 → 결제 실패/취소 → "결제에 실패했습니다" 메시지 + 재시도 버튼 → 재시도 클릭 → 결제창 재호출

### L. 관리자 문자 재발송 플로우

관리자 대시보드 → 주문 관리 → 주문 상세 → [문자 재발송] 클릭 → 확인 모달 ("기존 임시 비밀번호가 폐기되고 새 비밀번호가 발송됩니다") → 확인 → 기존 임시 비밀번호 폐기 + 새 임시 비밀번호 생성 + 타이머 20분 리셋 → 알리고 문자 발송 (바우처 URL + 새 임시 비밀번호) → 발송 이력에 기록

### M. 관리자 주문 취소 플로우

관리자 대시보드 → 주문 관리 → 주문 상세 (비밀번호 미설정 건) → [관리자 주문 취소] 클릭 → 취소 사유 선택 → 확인 모달 → PG 취소 API 호출 → 바우처 cancelled + 핀 재고 복구 → 구매자에게 취소 SMS 발송

### N. 자동 품절 / 재고 복구 플로우

핀 번호 대기 재고 0 도달 → 해당 상품 사용자 화면에서 자동 품절 표시 (구매 버튼 비활성화) → 관리자가 핀 번호 추가 등록 (개별/CSV) → 대기 재고 1 이상 → 자동으로 판매 상태 복구 (구매 버튼 활성화)

## 10. 비기능 요구사항

### 10-1. 보안

| 항목 | 요구사항 |
|------|---------|
| 통신 암호화 | 모든 통신 HTTPS (TLS 1.2 이상) |

| XSS 방지 | React 기본 이스케이핑 + CSP 헤더 설정 |
| 관리자 인증 | 관리자 전용 로그인, 세션/토큰 기반 인증 |

## 11. 용어 정의

| 용어 | 정의 |
|------|------|
| 바우처 (Voucher) | 구매 완료 후 발급되는 교환권. 고유 코드 + URL을 가짐. **1 주문 = 1 바우처.** 수량 N개 구매 시 바우처 1개에 핀 N개가 포함됨 |
| 바우처 코드 | 바우처를 식별하는 고유 랜덤 문자열. URL 경로에 사용 |
| 임시 비밀번호 | 구매/선물 시 SMS로 발송되는 3자리 영숫자. 최초 인증용 |
| 사용자 비밀번호 | 바우처 소유자가 설정하는 4자리 숫자. 핀 조회/선물 시 필요 |
| 핀 (PIN) 번호 | 실제 상품권의 핀 코드(숫자 16자리). 관리자가 사전 등록. AES-256 암호화 저장. 바우처 1개에 N개의 핀이 포함될 수 있음 |
| 수수료 포함 | 구매 시 상품 가격 + 수수료를 함께 결제. 핀 즉시 확인 가능 |
| 수수료 별도 | 구매 시 상품 가격만 결제. 핀 확인 시 수수료 별도 결제 필요 |
| 자동 품절 | 핀 대기 재고가 0이면 사용자 화면에서 자동으로 품절 표시 |
| 선물하기 | 바우처를 다른 회원에게 양도. 기존 바우처 비활성화 + 새 바우처 발급. 포함된 모든 핀이 함께 이동 |
