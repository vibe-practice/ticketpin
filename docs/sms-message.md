# SMS 발송 메시지 정의

> SMS 개발 태스크: P2-020 (알리고 SMS 연동)
> 이 문서의 메시지 템플릿을 그대로 사용하여 구현한다.

---

## 공통 규칙

| 규칙 | 설명 |
|------|------|
| 이모지 금지 | 모든 문자 메시지에 이모지 사용 금지, 텍스트만 구성 |
| 발신번호 | 사전 등록된 발신번호 사용 (환경변수 `ALIGO_SENDER`) |
| 메시지 유형 | 90byte 이하 SMS, 초과 시 LMS 자동 전환 |
| 재시도 | 발송 실패 시 최대 3회 자동 재시도 (1초 / 5초 / 30초 간격) |
| 로그 | 모든 발송은 `sms_logs` 테이블에 기록 (성공/실패 모두) |

---

## 발송 유형 목록

| 유형 코드 | 발송 시점 | 수신자 | 트리거 |
|-----------|-----------|--------|--------|
| `purchase` | 상품권 구매 완료 | 구매자 | 결제 승인 콜백 처리 후 |
| `reissue` | 임시 비밀번호 재발행 | 바우처 소유자 | 재발행 API 호출 시 |
| `gift` | 선물 수신 | 선물 받는 사람 | 선물 전송 API 완료 후 |
| `cancel` | 결제 취소 완료 | 구매자 | PG 취소 API 성공 후 |
| `admin_resend` | 관리자 문자 재발송 | 바우처 소유자 | 관리자가 주문 상세에서 재발송 실행 |

## 메시지 템플릿

### 1. 구매 완료 — 수수료 포함 (`purchase`)

```
[티켓핀] 상품권 구매 완료

{상품명} {수량}매
결제금액: {총 결제금액}원

아래 링크에서 상품권을 확인하세요.
{바우처 URL}

임시 비밀번호: {임시 비밀번호 3자리}
(유효시간 20분)

결제 취소는 구매 당일 자정까지만 가능합니다.
```

**변수 설명:**

| 변수 | 출처 | 예시 |
|------|------|------|
| `{상품명}` | `products.name` | 컬쳐랜드 상품권 1만원권 |
| `{수량}` | `orders.quantity` | 2 |
| `{총 결제금액}` | `orders.total_amount` (천 단위 콤마) | 21,000 |
| `{바우처 URL}` | `https://도메인/v/{vouchers.code}` | https://ticketpin.kr/v/a1b2c3d4-... |
| `{임시 비밀번호 3자리}` | 서버 생성 (영숫자 3자리) | k7m |

---

### 2. 구매 완료 — 수수료 별도 (`purchase`)

```
[티켓핀] 상품권 구매 완료

{상품명} {수량}매
결제금액: {총 결제금액}원

아래 링크에서 상품권을 확인하세요.
{바우처 URL}

임시 비밀번호: {임시 비밀번호 3자리}
(유효시간 20분)

핀 번호 해제 수수료: {수수료 총액}원
수수료 결제 후 핀 번호를 확인하실 수 있습니다.

결제 취소는 구매 당일 자정까지만 가능합니다.
```

**추가 변수:**

| 변수 | 출처 | 예시 |
|------|------|------|
| `{수수료 총액}` | `orders.fee_amount × orders.quantity` (천 단위 콤마) | 1,400 |

### 3. 임시 비밀번호 재발행 (`reissue`)

```

(유효시간 20분)

```

### 4. 선물 수신 — 수수료 포함 (`gift`)

```
[티켓핀] 상품권 선물 도착

{보낸 사람 이름}({보낸 사람 아이디})님이 상품권을 보냈습니다.

{상품명} {수량}매
금액: {상품 단가}원

아래 링크에서 상품권을 확인하세요.
{새 바우처 URL}

임시 비밀번호: {임시 비밀번호 3자리}
(유효시간 20분)
```

**변수 설명:**

| 변수 | 출처 | 예시 |
|------|------|------|
| `{보낸 사람 이름}` | `users.name` (복호화) | 홍길동 |
| `{보낸 사람 아이디}` | `users.username` | hong123 |
| `{새 바우처 URL}` | 선물 시 새로 생성된 바우처 코드 | https://ticketpin.kr/v/x9y8z7w6-... |
| `{상품 단가}` | `orders.product_price` (천 단위 콤마) | 10,000 |

### 5. 선물 수신 — 수수료 별도 (`gift`)

```
[티켓핀] 상품권 선물 도착

{보낸 사람 이름}({보낸 사람 아이디})님이 상품권을 보냈습니다.

{상품명} {수량}매
금액: {상품 단가}원

아래 링크에서 상품권을 확인하세요.
{새 바우처 URL}

임시 비밀번호: {임시 비밀번호 3자리}
(유효시간 20분)

핀 번호 해제 수수료: {수수료 총액}원
수수료 결제 후 핀 번호를 확인하실 수 있습니다.
```

---

### 6. 결제 취소 완료 (`cancel`)

```
[티켓핀] 결제 취소 완료

주문번호: {주문번호}
상품명: {상품명} {수량}매
취소금액: {환불 금액}원

카드사 사정에 따라 영업일 기준 3~5일 내에 취소가 반영됩니다.
```

**변수 설명:**

| 변수 | 출처 | 예시 |
|------|------|------|
| `{주문번호}` | `orders.order_number` | ORD-20260301-001 |
| `{환불 금액}` | `cancellations.refund_amount` (천 단위 콤마) | 21,000 |

**환불 금액 계산:**
- 수수료 포함 주문: `(product_price + fee_amount) × quantity` (전액)
- 수수료 별도 주문: `product_price × quantity` (수수료 미결제 상태이므로 상품 금액만)

---

### 7. 관리자 문자 재발송 (`admin_resend`)

```
[티켓핀] 임시 비밀번호 안내

아래 링크에서 상품권을 확인하세요.
{바우처 URL}

새 임시 비밀번호: {임시 비밀번호 3자리}
(유효시간 20분)
```

**처리 로직:**
- 기존 임시 비밀번호 폐기
- 새 임시 비밀번호 생성 + bcrypt 해싱 저장
- 타이머 20분 리셋 (`temp_password_expires_at` 갱신)
- 임시 비밀번호 시도 횟수 초기화
- 관리자 재발송은 재발행 5회 제한에 포함되지 않음

---

## 발송 조건 요약

| 유형 | 선행 조건 | 실패 시 |
|------|-----------|---------|
| `purchase` | PG 결제 승인 성공 + 바우처 생성 완료 | 3회 재시도, 실패 로그 기록 |
| `reissue` | 재발행 횟수 5회 미만 + 타이머 만료 상태 | 3회 재시도, 실패 로그 기록 |
| `gift` | 선물 트랜잭션 완료 (새 바우처 생성 + 핀 이동) | 3회 재시도, 실패 로그 기록 |
| `cancel` | PG 취소 API 호출 성공 | 3회 재시도, 실패 로그 기록 |
| `admin_resend` | 관리자 인증 + 대상 바우처 존재 | 3회 재시도, 실패 로그 기록 |

## sms_logs 테이블 기록 항목

| 컬럼 | 설명 |
|------|------|
| `message_type` | 위 유형 코드 (`purchase` / `reissue` / `gift` / `cancel` / `admin_resend`) |
| `message_content` | 실제 발송된 메시지 전문 |
| `send_status` | `pending` → `sent` 또는 `failed` |
| `sent_by` | `system` (자동) 또는 `admin` (관리자 재발송) |
| `aligo_response` | 알리고 API 응답 원본 (JSONB) |
