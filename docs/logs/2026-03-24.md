# 작업 로그 — 2026-03-24

## 01:20 — 블랙 & 화이트 모노톤 디자인 시스템 전환 + 메인/카테고리/상품상세/주문/인증 페이지 재디자인

### 요약 (비개발자용)
> 티켓핀 프로젝트의 전체 디자인을 퍼플 테마에서 블랙 & 화이트 모노톤으로 전면 변경했습니다.
> 디자인 토큰(색상, 폰트 등)을 새로 정의하고, 메인 홈페이지의 레이아웃을 사이드바 방식에서
> 상단 헤더 + 우측 사이드바(로그인/고객센터) 구조로 바꿨습니다. 카테고리 목록, 상품 상세,
> 주문/결제, 인증(로그인/회원가입/아이디찾기/비밀번호재설정) 페이지도 모노톤 스타일로 재디자인했습니다.
> 디자인 시스템 스킬의 reference 폴더 문제도 발견하여 수정했습니다.

### 작업 내용

**1. 디자인 시스템 전환**
- globals.css: 퍼플 → 블랙/뉴트럴 그레이 토큰으로 전면 교체
- DESIGN-SYSTEM.md: 모노톤 디자인 시스템 문서 재작성
- 폰트 컬러를 reference 고정값으로 수정 (#111111, #505050, #767676, #999999)
- 전체 프로젝트 85개 파일에서 violet/purple/amber/slate 클래스 제거
- 29개 파일에서 text-neutral-* → 시맨틱 클래스 4가지(text-foreground, text-secondary-foreground, text-muted-foreground, text-white)로 통일

**2. 레이아웃 변경**
- 사이드바(Sidebar.tsx) + 탑바(TopBar.tsx) → SiteHeader.tsx (상단 GNB + 검색창) 전환
- Sidebar.tsx, TopBar.tsx 삭제 (Dead Code)
- SiteLayout.tsx: 헤더 항상 고정 (스크롤 숨김 제거)
- 헤더에 카테고리 메뉴 제거 → 메인 콘텐츠 안에 CategoryNav 컴포넌트로 이동
- 헤더에 검색창 길게 배치

**3. 메인 홈페이지 재디자인**
- 2컬럼 레이아웃 (메인 콘텐츠 + 우측 사이드바 280px sticky)
- HeroBanner: 메인 콘텐츠 안 카드형 캐러셀 (좌우 버튼, 5초 자동, 인디케이터)
- HomeSidebar: 로그인 카드 + 프로모션 배너 + 고객센터 정보
- CategoryGrid: 가로 스크롤, 아이콘 88px, 6개/줄, 좌우 버튼
- ProductGrid: 4개/줄 가로 스크롤, 좌우 스와이프 버튼 (항상 표시, 스크롤 가능 시 활성)
- NoticePreview: 실제 공지사항 API 연동
- 이벤트 특가/BEST 추천상품 섹션 삭제 → 인기 상품 + 카테고리별 상품으로 단순화
- Footer: 4컬럼 링크 그리드 + 고객센터 + 회사정보 리디자인
- 텍스트 전체 14px 이상으로 통일

**4. 카테고리 목록 페이지**
- 브레드크럼 + 카테고리 탭(가로 스크롤) + 정렬 + 재검색 + 4개/줄 그리드 + 페이지네이션
- 전체/카테고리별 동일 레이아웃 공유

**5. 상품 상세 페이지**
- 2컬럼 (이미지 50% + 정보 50%), 수수료 선택 카드, 수량 선택, 탭 (설명/유의사항/환불)
- 수수료 카드 활성 시 텍스트 text-white/70으로 가시성 개선
- 보더 neutral-100 → neutral-300 변경

**6. 주문/결제 페이지**
- 2컬럼 (주문정보 + 결제금액 사이드), container-main 적용
- 주문완료, 결제승인, 결제취소 페이지 모노톤 재디자인

**7. 인증 페이지 4개**
- 로그인, 회원가입, 아이디찾기, 비밀번호재설정 모노톤 재디자인
- "티켓핀" 로고 제거, 제목/설명 중앙 정렬
- 기존 비즈니스 로직 (RHF+Zod, API, Zustand) 완전 유지

**8. 디자인 시스템 스킬 수정**
- reference/ (단수) vs references/ (복수) 폴더 불일치 발견
- reference/ 30개 파일 → references/로 복사, reference/ 삭제
- 배너/이미지 관리 태스크 추가 (TASKS-BANNER.md)
