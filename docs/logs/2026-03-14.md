## 02:44 — [이슈 #2] 수수료 이중결제 망취소 실패 수정

### 요약 (비개발자용)
> 수수료 결제가 네트워크 문제 등으로 두 번 들어갔을 때, 시스템이 두 번째 결제를
> 자동으로 취소하려 합니다. 그런데 이 자동 취소마저 실패하면 고객 카드에서 돈이
> 두 번 빠진 채로 방치되고, 시스템은 "잘 됐다"고 거짓 응답을 보내는 문제가
> 있었습니다.
>
> 이번 수정으로:
> - 자동 취소가 실패하면 최대 3번까지 재시도합니다
> - 그래도 실패하면 "이중결제가 발생했습니다"라는 에러 메시지를 보여줍니다
> - 실패 기록이 관리자 화면에 남아서 관리자가 수동으로 재시도할 수 있습니다
> - 정산 페이지에 경고 배너가 떠서 관리자가 바로 인지할 수 있습니다

### 작업 내용
- `executePgReversal` 함수 반환타입을 `void` → `PgReversalResult`로 변경하여 성공/실패 여부를 호출부에 전달
- 망취소 자동 재시도 로직 추가 (최대 3회, 500ms → 1s → 2s 지수 백오프)
- 망취소 실패 시 `cancellations` 테이블에 `reason_type: "duplicate_payment"`, `refund_status: "failed"` 자동 기록하는 `recordReversalFailure` 헬퍼 추가
- 관리자 retry API의 stub을 실제 `cancelPgPayment()` 호출로 교체
- `CancellationReasonType`에 `"duplicate_payment"`, `CancelledBy`에 `"system"` 타입 추가
- `Cancellation` 인터페이스에 `pg_ref_no`, `pg_tran_date`, `pg_pay_type` 필드 추가 (retry 시 원 결제 정보 참조용)
- 관리자 환불 관리 페이지에 이중결제/시스템 라벨 및 필터 옵션 추가
- 정산 전체보기 페이지에 미처리 이중결제 경고 배너 추가
- 코드 리뷰에서 발견된 `cancelled_by` 필터 누락, PG 정보 null 시 false positive 등 3건 추가 수정
