## 00:13 — Supabase 실제 데이터 연동 + API/UI 개선 + 사이드바 버그 수정

### 요약 (비개발자용)
> 정산 전체보기 페이지에 실제 데이터베이스를 연결하여 더미 데이터 대신
> 진짜 업체/정산 정보가 표시되도록 했습니다.
>
> 코드 리뷰를 통해 발견된 성능 문제 3가지를 수정했습니다:
> 불필요한 데이터베이스 쿼리 제거, 미사용 필드 정리, 날짜 변경 시
> API가 두 번 호출될 수 있는 문제 해결.
>
> 또한 모바일에서 날짜 필터가 한 줄에 안 들어가 깨지는 문제,
> 날짜 클릭 영역이 좁은 문제, 그리고 "전체보기"를 눌렀는데
> "정산 관리" 메뉴도 함께 활성화되는 사이드바 버그를 수정했습니다.

### 작업 내용
- `GET /api/admin/settlements/overview` API 신규 생성 (Supabase 연동)
  - businesses 전체 조회 + 기간별 settlements 집계
  - 업체별 대표 정산 상태 우선순위 로직 (pending > confirmed > paid > cancelled)
- 프론트엔드 더미 데이터 제거, API fetch + 로딩/에러 처리
- 코드 리뷰 반영 (경고 3건):
  - `.in(bizIds)` 불필요 필터 제거 (업체 100곳 규모 대비)
  - `account_holder` 미사용 select 제거
  - 날짜 상태를 `dateRange` 단일 객체로 통합 (이중 API 호출 방지)
  - 상태 우선순위 상수를 루프 밖으로 추출 + 비즈니스 의미 주석 추가
- 모바일 기간 필터 레이아웃 수정 (flex-1 + md:flex-none 반응형)
- 날짜 input 전체 영역 클릭 시 `showPicker()` 호출
- 사이드바 서브메뉴 활성화 로직 수정 (`isChildActive` — prefix 충돌 방지)
